# Platform Philosophy

Status: Stable
Doc Type: Philosophy

FloorConnector is a specialty contractor operating system, not a collection of module islands.

It is built for epoxy flooring, concrete polishing, resinous flooring, and specialty surface contractors who need continuity across sales, operations, execution, finance, customer interaction, and future growth layers.

## Core Philosophy

- Continuity over silos.
- Project chain over module islands.
- Canonical records over duplicated entry.
- Customer and contractor surfaces on one shared truth.
- Provider integrations as adapters.
- AI as an operating layer, not a parallel system.

## Expansion Rule

Every future expansion should strengthen the shared workflow:

`opportunity -> customer -> project -> estimate -> contract -> change order -> job -> invoice -> payment`

If a feature cannot explain how it attaches to that chain, it is probably drifting toward a silo.
