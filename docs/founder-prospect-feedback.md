# Founder Prospect Feedback

Status: Active
Doc Type: Demo Operations

Use this worksheet after each controlled founder-prospect demo. It is a feedback capture tool, not a product commitment, pricing quote, support ticket system, or implementation plan.

Use with [docs/founder-prospect-demo-script.md](C:/FloorConnector/docs/founder-prospect-demo-script.md), [docs/founder-demo-readiness.md](C:/FloorConnector/docs/founder-demo-readiness.md), [docs/paid-early-access-plan.md](C:/FloorConnector/docs/paid-early-access-plan.md), and [docs/saas-billing-live-launch-plan.md](C:/FloorConnector/docs/saas-billing-live-launch-plan.md).

## Capture Rules

- Capture notes the same day as the demo.
- Do not include passwords, env values, invite tokens, raw invite links, Checkout URLs, Stripe keys, webhook secrets, service-role keys, payment details, raw webhook payloads, or private customer data.
- Record the prospect's words when possible, but remove identifying customer names or sensitive project details before sharing.
- Separate observed friction from promised follow-up.
- Choose one recommended next build slice only after completing the worksheet.

## Decision Notes

Use these interpretation rules:

- If the prospect focuses on proposal accuracy, reusable systems, scope clarity, or material/labor assumptions, choose estimate/catalog/materials depth.
- If the prospect focuses on crew timing, signed-work handoff, capacity, or job planning, choose scheduling/dispatch depth.
- If the prospect likes the workflow but struggles to scan or use it quickly, choose manager/mobile polish.
- If the prospect wants to start but cannot bring data in or trust they can leave with data, choose import/export readiness.
- If the owner cannot evaluate business health from the app, choose reporting/dashboard depth.
- If the prospect is ready to pay and the blocker is controlled subscription operations, choose live billing readiness controls.
- If the product value is clear but first-run positioning or onboarding is too hard to repeat, choose onboarding/marketing polish.
