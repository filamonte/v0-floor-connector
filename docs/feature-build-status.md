# Feature Build Status

Status: Active
Doc Type: Status Inventory

This is a living status map for FloorConnector features. It is meant to support
investor prep, demo planning, product planning, and developer handoff without
turning roadmap direction into a sales promise.

Use [docs/current-state.md](C:/FloorConnector/docs/current-state.md) as the
source of truth for implemented capability. This document is a curated status
inventory above current-state, not a competing truth source. Use
[docs/Roadmap.md](C:/FloorConnector/docs/Roadmap.md),
[docs/platform-build-registry.md](C:/FloorConnector/docs/platform-build-registry.md),
and
[docs/platform-maturity-model.md](C:/FloorConnector/docs/platform-maturity-model.md)
for sequencing and maturity discipline. If this document and current-state ever
conflict, trust current-state until this document is updated.

Canonical lifecycle:

`opportunity -> customer -> project -> estimate -> contract -> change order -> job -> invoice -> payment`

Status vocabulary used here:

- Implemented: current code/docs prove the feature exists on the current branch.
- Partially implemented: a foundation, route, read model, or narrow workflow
  exists, but production depth remains.
- Planned: documented near- or mid-term work, not current shipped capability.
- Strategic future: long-term product direction, not current implementation.
- Docs-only: documented doctrine, audit, plan, or runbook without product
  runtime changes.
- Blocked / needs decision: owner, policy, provider, data, auth, or architecture
  decision is needed before implementation can proceed.
