# Contractor Foreman Gap Decision List

Status: Active
Doc Type: Roadmap / Product Direction

This document captures owner feature-coverage decisions from the Contractor Foreman baseline comparison. Contractor Foreman is a reference point for common contractor-system coverage, not the FloorConnector destination.

For implemented truth, use [docs/current-state.md](C:/FloorConnector/docs/current-state.md), [docs/module-status.md](C:/FloorConnector/docs/module-status.md), and [docs/floorconnector-full-capability-audit.md](C:/FloorConnector/docs/floorconnector-full-capability-audit.md). This document records product direction and must not be read as implementation status.

Canonical lifecycle to preserve:

`opportunity -> customer -> project -> estimate -> contract -> change order -> job -> invoice -> payment`

## Product Rule

FloorConnector should cover the serious contractor operating needs that products like Contractor Foreman prove are common, then go deeper for epoxy flooring, resinous flooring, polished concrete, coatings, and specialty surface contractors. Future capabilities must attach to canonical records and must not become detached modules, portal-only copies, AI-only truth, or duplicate business models.

## Deferred Or Not-Core Decisions

- Permits are low priority and should remain a project compliance/checklist/document concept if added.
- Earned value is desired but later, after job costing, progress billing, labor, materials, equipment, and production tracking mature.
- Generic detached to-dos are not core; record-linked work items and deterministic cues are the right direction.
- GPS time verification is future; the immediate need is a real canonical clocking/time-card system.
- QuickBooks/accounting integration is definite later work, not the source of truth for FloorConnector financial records.
